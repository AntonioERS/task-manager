Atributos globales
Los atributos globales puede ser especificados en todos los elementos HTML
download
Hace que el navegador trate el enlace como una descarga. Puede ser utilizado con o sin el valor filename. Sin el valor, el navegador sugerirá el nombre y extensión, con filename el valor será el sugerido como nombre de archivo. Solamente funciona con URLs locales
href
La URL al que apunta el hipervínculo. Puede ser otra página, una sección del propio documento (utilizando # para indicar el id de la sección)
target
Donde mostrar la URL, con los valores _self (por defecto, la propia página), _blank (normalmente una nueva pestaña), _parent (el contexto padre de navegación), _top (el contexto "más alto" de navegación)

alt
Contiene una descripción en texto de la imagen, útil para accesibilidad
height
La altura intrínseca de la imagen, en píxeles. Debe ser un entero sin unidades. Es útil especificarla para que el navegador "reserve" el sitio de la imagen al cargar la página
src
Atributo obligatorio, contiene el vínculo a la imagen
srcset
Una o varias cadenas separadas por comas, indicando las posibles fuentes de la imagen en caso de utilizar distintos tamaños o resoluciones. Cada cadena está compuesta de una URL a la imagen y (opcionalmente) un espacio en blanco seguido de: o un indicador de la anchura (entero positivo seguido de w) o un indicador de la densidad de pixeles (número decimal positivo seguido de x). No es posible utilizar ambos indicadores al mismo tiempo

La anchura intrínseca de la imagen, en píxeles. Debe ser un entero sin unidades. Es útil especificarla para que el navegador "reserve" el sitio de la imagen

<time>

Representa un período específico de tiempo.

Puede incluir el atributo datetime para traducir fechas en formato legible para el navegador, permitiendo mejores resultados para los motores de búsqueda o para recordatorios. Puede representar uno de los siguientes:

  • Una hora en un reloj de 24 horas
  • Una fecha concreta en el calendario gregoriano (opcionalmente con información de hora y zona horaria)
  • Una duración válida de tiempo

<time datetime="2020-05-23" pubdate>Publicado el 23-05-2020</time>

Indica la hora y/o fecha del elemento, y debe estar en uno de los siguientes formatos:

  • Una cadena de fecha válida: 2011-11-18
  • Una cadena de fecha sin año válida: 11-18
  • Una cadena de semana válida: 2011-W47
  • Una cadena de hora válida: 14:54 14:54:39 14:54:39.929
  • Una cadena de fecha y hora local válida: 2011-11-18T14:54:39.929 2011-11-18 14:54:39.929
  • Una cadena de fecha y hora global válida: 2011-11-18T14:54:39.929Z 2011-11-18T14:54:39.929-0400 2011-11-18T14:54:39.929-04:00 2011-11-18 14:54:39.929Z 2011-11-18 14:54:39.929-0400 2011-11-18 14:54:39.929-04:00

<audio>

Para insertar audio

Puede contener una o más fuentes de audio utilizando el atributo src o el elemento source, eligiendo el navegador la más adecuada. También puede ser el destinatario de medios transmitidos. El contenido entre las etiquetas <audio></audio> es el que se muestra como respaldo en los navegadores que no soportan el elemento.

<audio controls> <source src="ejemploAudio.mp3" type="audio/mp3"> </audio>

Atributos globales
Los atributos globales puede ser especificados en todos los elementos HTML
autoplay
Atributo booleano. Si es especificado, el audio comenzará a reproducirse automáticamente tan pronto como sea posible, sin esperar a que se cargue todo el archivo de audio. Práctica no recomendada.
controls
Si está presente, el navegador ofrecerá controles para la reproducción, volumen, búsqueda y pausar/continuar la reproducción. Si no se incluye, el navegador no mostrará los controles por defecto.
controlslist
Experimental. Ayuda al navegador a mostrar los controles adecuados al audio, cuando se ha especificado el atributo controls. Los valores admitidos son: nodownload, nofullscreen y noremoteplayback.
crossorigin
Indica cuándo utilizar CORS (Cross-Origin Resouce Sharing) para buscar el archivo de audio. Los valores admitidos son anonymous y use-credentials. Si no está presente, el recurso es buscado sin una petición CORS.
disableremoteplayback
Experimental. Atributo booleano utilizado para desactivar la capacidad de reproducción remota de dispositivos conectados via cable (HDMI, DVI, etc) o inalámbrica (Miracast, Chromecast, AirPlay, etc)
loop
Atributo booleano, si está especificado el reproductor de audio volverá al inicio al terminar de reproducir el audio.
muted
Atributo booleano, indica si el audio estará inicialmente desactivado. Su valor por defecto es false.
preload
Su intención es dar una pista al navegador sobre lo que el autor piensa que será la mejor experiencia para el usuario. Los valores admitidos son:
  • none Indica que el audio no debe ser precargado.
  • metadata Indica que sólo los metadatos (por ejemplo, longitud) del audio son cargados.
  • auto Indica que todo el fichero de audio puede ser cargado, incluso si el usuario no espera usarlo.
  • Cadena vacía: Un sinónimo del valor auto
El valor por defecto es distinto para cada navegador. Se recomienda utilizar el valor metadata
src
La URL del audio a incrustar. Es opcional, en su lugar puede utilizarse el elemento <source> con varias fuentes distintas que den soporte a distintos navegadores.

<video>

Para insertar video

También puede insertar audio, útil para cuando queremos añadir una transcripción del audio, en los demás casos el elemento <audio> ofrece una mejor experiencia al usuario. El contenido entre las etiquetas <video></video> es el que se muestra como respaldo en los navegadores que no soportan el elemento.

Para mostrar subtítulos se puede emplear JavaScript con el elemento <track> y el formato WebVTT.

<video controls width="250" src="ejemploVideo.webm"> <a href="ejemploVideo.webm">Descarga el video en webm</a> </video>

Los atributos globales puede ser especificados en todos los elementos HTML
autoplay
Atributo booleano. Si es especificado, el video comenzará a reproducirse automáticamente tan pronto como sea posible, sin esperar a que se cargue todo el archivo de video. Práctica no recomendada.

Utiliza el atributo disablepictureinpicture si quieres desactivar el modo y el control de Imagen a Imagen.
crossorigin
Indica cuándo utilizar CORS (Cross-Origin Resouce Sharing) para buscar el archivo de video. Los valores admitidos son anonymous y use-credentials. Si no está presente, el recurso es buscado sin una petición CORS.

Experimental. Evita que el navegador sugiera el menú contextual de Imagen a Imagen.
disableremoteplayback

La altura del área para mostrar el vídeo, en píxeles CSS (sólo valores absolutos, no porcentajes). Se puede usar la propiedad de CSS object-position para ajustar la posición del vídeo dentro del marco del elemento, y la propiedad object-fit para controlar cómo el tamaño del vídeo se ajusta para encajar en el marco.
loop
Atributo booleano, si está especificado el reproductor de video volverá al inicio al terminar de reproducir el video.
muted
Atributo booleano, indica si el audio del vídeo estará inicialmente desactivado. Su valor por defecto es false (el audio se reproducirá cuando se reproduzca el video).
playsinline
Atributo booleano indicando que el vídeo será reproducido "en línea", en el interior del área de reproducción del elemento. La ausencia de este elemento no implica que el vídeo será reproducido siempre a toda pantalla.
poster
Una URL de una imagen para mostrar mientras el vídeo está cargando. Si no se especifica este atributo, no se mostrará nada hasta que se cargue el primer fotograma, que entonces pasará a mostrarse.

  • Cadena vacía: Un sinónimo del valor auto
El valor por defecto es distinto para cada navegador. Se recomienda utilizar el valor metadata
src
La URL del video a incrustar. Es opcional, en su lugar puede utilizarse el elemento <source> con varias fuentes distintas que den soporte a distintos navegadores.
width
El ancho del área para mostrar el vídeo, en píxeles CSS (sólo valores absolutos, no porcentajes). Se puede usar la propiedad de CSS object-position para ajustar la posición del vídeo dentro del marco del elemento, y la propiedad object-fit para controlar cómo el tamaño del vídeo se ajusta para encajar en el marco.

reversed
Atributo booleano que especifica que los elementos de la lista están en orden invertido. Los elementos serán numerados de mayor a menor.
start
Entero desde el que empieza a contar la lista de elementos. Siempre un número arábigo, incluso cuando el atributo type es letras o numerales romanos. Por ejemplo, para empezar a numerar elementos desde la letra d o el numeral romano IV, utiliza start="4"
type

Establece el tipo de numeración:

  • a para letras en minúscula
  • A para letras en mayúscula
  • i para números romanos en minúscula
  • I para números romanos en mayúscula
  • 1 para números (por defecto)

El tipo especificado se usa en toda la lista a no ser que un atributo type distinto se use en un elemento <li>

A no ser que el tipo de número de la lista importe (como en documentos legales o técnicos donde se refiere a los elementos por su número/letra), utiliza en su lugar la propiedad de CSS list-style-type

Representa seis niveles de encabezado de texto, siendo <h1> el más alto y <h6> el más bajo. No tiene más atributos que los atributos globales.

Tabla de atributos globales
Atributo Descripción
accesskey Da una sugerencia para generar un atajo de teclado al elemento actual. Este atributo consiste en una lista de caracteres separados por un espacio. El navegador debería utilizar el primero que exista en el teclado.
autocapitalize

Controla cuando y cómo la entrada de texto es automáticamente puesta en mayúsculas. Puede tener los siguientes valores:

  • off o none, no se pone en mayúsculas automáticamente (todas las letras por defecto en minúsculas)
  • on o sentences, la primera letra de cada frase se pone por defecto en mayúsculas, todas las demás letras en minúsculas.
  • words, la primera letra de cada palabra se pone por defecto en mayúsculas, todas las demás letras en minúsculas.
  • characters, todas las letras por defecto en mayúsculas.
autofocus Indica que es un elemento en el cae el foco al cargar la página, o tan pronto como el <dialog> del que es parte se muestra. Es un atributo booleano, inicialmente falso.
class Lista de clases a las que pertenece el elemento, separadas por un espacio. Estas clases permiten que CSS y JavaScript accedan a esos elementos específicos
contenteditable

Indica si el elemento debería ser editable por el usuario. Si lo es, el navegador modifica su comportamiento para permitir la edición. Puede tomar los siguientes valores:

  • true o cadena vacía, lo que el indica que el elemento debe ser editable.
  • false lo que indica que el elemento no debe ser editable
data-* Forma una clase de atributos, llamada datos de atributos a medida. Utilizada por scripts.
dir

Indica la dirección del texto del elemento. Puede tener los siguientes valores:

  • ltr, que significa izquierda a derecha, y es usado por idiomas que son escritos de izquierda a derecha.
  • rtl, que significa de derecha a izquierda, y es usado por idiomas que son escritos de derecha a izquierda, com el árabe.
  • auto, que permite que el navegador elija.
draggable

Indica si el elemento puede ser arrastrado, utilizando la API Drag an Drop. Puede tener los siguientes valores:

  • true, que indica que el elemento puede ser arrastrado.
  • false, que indica que el elemento no puede ser arrastrado.

enterkeyhint Sugiere el icono o la etiqueta de acción a utilizar como tecla enter en teclados virtuales.
exportparts Experimental. Usado para exportar transitivamente partes de sombra desde un árbol de sombras anidiado a un árbol que contiene luces.
hidden Indica que el elemento todavía no es (o ya no es) relevante. Por ejemplo, puede utilizarse para ocultar elementos de la página que no pueden usarse hasta que el proceso de login ha sido completado. El navegador no renderizará dichos elementos.
id Define un identificador único en todo el documento, permitiendo identificar ese elemento desde CSS y JavaScript
inert Un valor booleano que hace que el navegador ignore eventos de entrada para el elemento. Útil cuando están presentes eventos click.
inputmode Sugiere a los navegadores la configuración del teclado virtual a usar cuando se edite este elemento o sus contenidos. Usado principalmente en elementos <input>
is Permite especificar que un elemento HTML estándar debe comportarse como un elemento registrado hecho a medida.
itemid Identificador global único de un ítem.
itemprop Utilizado para añadir propiedades a un ítem.
itemref Las propiedades que no son descendientes de un elemento con el atributo itemscope pueden ser asociadas a un ítem utilizando itemref. Proporciona una lista de ids con propiedades adicionales en cualquier lugar del documento.
itemscope Generalmente funciona con itemtype para especificar que el HTML contenido en un bloque es acerca de un ítem en particular.
itemtype Especifica la URL del vocabulario que será utilizado para definir itemprops (propiedades del ítem) en la estructura de datos.
lang Ayuda a definir el idioma de un elemento: el idioma en que están elementos no editables o el idioma en que el usuario debería escribir los elementos editables. El atributo contiene una etiqueta de lenguaje en el formato definido en RFC 5646.
nonce Un nonce (number used once) empleado con propósitos criptográficos.
part Una lista de nombres part del elemento separados por espacios. Los nombres part permite que CSS seleccione y de estilo a elementos específicos del árbol de sombras via el pseudo elemento ::part.
popover Especifica que un elemento debe ser tratado como un pop-over. Un elemento con el atributo popover permanece oculto a la página a no ser que sea abierto interactuando con un elemento invocador que tenga popovertarget, o via showPopover()
role Define el valor semántico de un elemento, permitiendo que lectores de pantalla y otras herramientas persenten y soporten interacción con un objeto de forma que sea consistente con las expectativas del usuario para ese tipo de objeto.
slot Asigna una ranura en árbol de sombras a un elemento.
spellcheck

Define si el elemento puede ser verificado para errores ortográficos. Puede tener los siguietes valores:

  • true o cadena vacía, que indica que el elemento debería ser, si es posible, verificado para errores ortográficos.
  • false, que indica que el elemento no debería ser verificado para errores ortográficos.
style Contiene declaraciones de estilo CSS para aplicar al elemento. Se recomienda que los estilos sean definidos en archivos independientes, el objetivo de este atributo es la aplicación rápida de un estilo, por ejemplo para hacer pruebas.
tabindex

Un entero indicando si el elemento puede ser enfocado, si debería ser parte de una navegación secuencial del teclado y, de serlo, en qué posición. Puede tomar varios valores:

  • Un valor negativo indica que el elemento puede ser enfocado, pero no debería incluirse en la navegación secuencial del teclado.
  • 0 significa que el elemento puede ser enfocado e incluido en la navegación secuencial del teclado, pero su orden relativo es definido por la convención de la plataforma.
  • Un valor positivo indica que el elemento puede ser enfocado e incluido en la navegación secuencial del teclado, el orden en que los elementos son enfocados es el valor creciente de tabindex. Si varios elementos tienen el mismo tabindex, su orden relativo sigue sus posiciones relativas en el documento.

title Contiene un texto representando información relacionada con el elemento al que pertenece.
translate

Utilizado para especificar si los valores de los atributos de un elemento y los valores de sus nódulos hijo Text son traducidos cuando la página es localizada, o si se dejan sin cambios. Puede tomar los siguientes valores:

  • yes o cadena vacía, que indica que el elemento será traducido.
  • no, que indica que el elemento no será traducido.
virtualkeyboardpolicy

Utilizado para controlar el comportamiento del teclado en pantalla en dispositivos como tablets, teléfonos móviles y otros dispositivos donde no podría estar disponible un teclado físico para elementos que también utilizan el atributo contenteditable. Puede tomar los siguientes valores:

  • auto o cadena vacía, lo que automáticamente muestra el teclado virtual cuando se enfoca el elemento o se toca.
  • manual, que desacopla el enfoque y el toque en el elemento del estado del teclado virtual.

Vocabulario

Elemento vacío (void element)
Elemento que no tiene nodos hijo (es decir, elementos anidados o textos). Sólo tienen etiqueta de apertura, no tienen etiqueta de cierre. Los elementos vacíos en HTML son los siguientes:
  • <area>
  • <base>
  • <br>
  • <col>
  • <embed>
  • <hr>
  • <img>
  • <input>
  • <link>
  • <meta>
  • <source>
  • <track>
  • <wbr>
